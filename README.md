# AIコンテンツ再利用ツール（日本向け・発信者向け）
## 要件定義 / プロンプト設計 / UI-UX 設計（MVP）

---

## 0. 前提・思想

- **本プロダクトは「AIツール」ではない**
- 役割は **「動画内容を、日本のSNSでそのまま投稿できる形に翻訳すること」**
- Claude / NotebookLM / n8n と競合しない
  - これらを使いこなせる層は対象外
  - 「考えたくない発信者」のためのプロダクト
- プロンプトや微調整は一切ユーザーに触らせない
- 出力は常に「完成形」であることを目指す

---

## 1. 想定ユーザー（MVP）

- 日本語で発信する個人YouTuber / ビジネス系発信者
- 動画制作はできるが、文章化やSNS向けの書き換えが苦痛・苦手
- X（旧Twitter）や note を主な発信媒体としている
- AIに対しては「便利そう」程度の認識で、Claude等のプロンプト操作は行わない

---

## 2. スコープ定義

### やること（MUST）

#### 入力
- YouTube URL（1本単位）

#### 選択項目（最大3つ・直感的な選択）
- **投稿タイプ**：解説・学び / 体験談 / ノウハウ共有
- **トーン**：やわらか / 普通 / ビジネス
- **絵文字**：あり / なし

#### 出力（固定・一括生成）
- **X スレッド**：そのままコピーして連投可能な形式
- **note 要約記事**：タイトル案付きの構成済み文章

---

### やらないこと（MUST NOT）

- SNSへの自動投稿（API連携による複雑化の回避）
- 短尺動画（TikTok / Instagram / Shorts）向けの出力
- カスタムプロンプト入力機能
- 生成パラメータ（温度・文字数等）の調整
- 高機能な編集エディタ（そのままコピペを前提とする）
- チーム機能・履歴管理・分析機能

---

## 3. 非機能要件（日本市場への最適化）

### 日本語・文脈のこだわり
- **謙虚なトーン**：断定しすぎず、上から目線にならない配慮
- **リスク回避**：炎上しにくい、マイルドかつ誠実な言い回し
- **可読性**：Xのタイムラインで目が止まる「改行・余白・リズム」の自動最適化
- **絵文字の質**：文末に機械的に置くのではなく、意味のある位置に配置

---

## 4. プロンプト設計（内部プロセス）

### 全体方針
- 1つのプロンプトで全てを処理せず、**「構造化 → 媒体別変換」**のステップを踏むことで精度を安定させる。

### 4-1. 内容構造化プロンプト（内部処理）
**目的：** 話し言葉のノイズを除去し、論理的な骨格を抽出する。

> **入力：** YouTube 文字起こし
> **役割：** 日本語コンテンツ編集者
> **出力項目：**
> - 動画のテーマ（1文）
> - 主張（1〜2文）
> - 理由・背景（箇条書き）
> - 具体例・エピソード（箇条書き）
> - 学び・ベネフィット（箇条書き）

### 4-2. Xスレッド生成プロンプト
**目的：** 日本のX文化に最適化された「読まれる」連投ツイートの作成。

> **入力：** 上記の構造化データ + ユーザー選択（タイプ/トーン/絵文字）
> **制約：**
> - 1ツイ目：引き（フック）を作るが、煽りすぎない。「〜と思った話」等の主観を入れる
> - 2〜5ツイ目：自然な文章で内容を解説。箇条書き感を出しすぎない
> - 最終ツイ：自然な動画誘導（売り込み感を排除）
> - 番号付け（1/n）の付与

### 4-3. note 要約生成プロンプト
**目的：** 記事単体で価値がありつつ、動画視聴へ繋げる要約記事の作成。

> **入力：** 構造化データ
> **構成：**
> 1. タイトル案（3つ）
> 2. 導入文（背景と問題意識）
> 3. 本文（適切な見出し設置）
> 4. まとめ
> 5. 動画リンクへの自然な誘導

---

## 5. UI / UX 設計（MVP）

### 基本方針
- **書かせない・考えさせない・迷わせない**
- 操作は 3 ステップ以内

### 画面①：入力（Landing 兼用）
- **キャッチコピー**：「動画を、投稿に変換する。」
- **メイン機能**：YouTube URL のペースト欄
- **オプション**：3つのセレクトボックス（タイプ/トーン/絵文字）
- **ボタン**：「投稿を作る」

### 画面②：出力
- **メッセージ**：「日本のSNS向けに調整済みです。そのまま投稿できます。」
- **Xスレッド表示エリア**：カード形式で表示。一括コピーボタンと1ツイートずつのコピーボタン。
- **note表示エリア**：タイトル案の選択と、本文のプレビュー・コピー。
- **再生成**：「言い回しを少し変える」ボタン（設定変更はさせない）

---

## 6. 排除する要素（NGリスト）
- 自由記述の編集エディタ
- プロンプトの露出
- AIの「思考プロセス」の長々とした表示
- 過去の履歴一覧（MVPではシンプルさを優先）

---

## 7. 成功指標
- 「Claudeを立ち上げて指示を考えるより、これを使った方が早い」という体験
- カスタマイズ要望よりも「このまま出せるのが楽」という声の最大化
